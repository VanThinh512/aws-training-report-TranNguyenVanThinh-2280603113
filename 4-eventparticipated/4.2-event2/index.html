Event 2

FCAJ Community Day – AI Context Engineering, CloudFront & Multi-Agent Systems

Event Content

The second Community Day featured seven sharing sessions covering market trends, AI prompt/context engineering, AWS product demos, CDN architecture, hackathon experiences, LLM internals, and enterprise multi-agent systems.

Session 1 – Market Trends & Personal Value (Speaker: Anh Hưng)

Anh Hưng shared insights on current and future market demands:

  • Understanding the evolving needs of the tech industry today and in the coming years.
  • The importance of building your own personal value — developing unique skills and expertise that set you apart in a competitive market.

Session 2 – Context Engineering for AI Prompts (Speaker: Anh Tịnh)

Anh Tịnh presented best practices for writing effective AI context to maximize productivity:

  • Give AI the context it needs — be precise about what you want, don’t add redundant information the model already knows.
  • Common mistake #1: Pulling everything from the internet and dumping it into context — this dilutes quality.
  • Common mistake #2: Telling the AI things it already knows. Don’t over-explain — state the problem to be solved next.
  • Shared a framework for writing standardized, effective context.
  • Introduced Obsidian as a “Second AI Brain” — using Obsidian as a personal knowledge management system to feed structured context to AI tools.

Session 3 – Friendly AI Assistant with Amazon Q (AWS Demo)

A demonstration of AWS’s platform connectivity ecosystem enabling users to self-build AI products:

  • AWS creates a platform connection across ecosystems so users can build their own products.
  • Demo 1: From a single Excel file, the AI analyzes insights and creates an agent based on user needs.
  • Demo 2: An agent that summarizes meeting content and automatically sends meeting notes to all participants via email.

Session 4 – CloudFront as Your Foundation

A deep-dive into Amazon CloudFront’s new capabilities and pricing model:

New Pricing Model
  • Target user groups: Small website owners, business users, scaling businesses
  • Prediction-based cost estimation
  • Easy plan selection and switching
  • Fixed-price CDN + security pricing plans
  • Cost savings strategies with CloudFront and reduced utilization
Security Capabilities
  • HTTPS support with encryption
  • Mutual TLS support for client certificate authentication
  • Origin Cloaking with VPC Origin — creating a tunnel from CloudFront directly into the origin
  • Origin Cloaking for Custom Origins — hiding origin servers from direct access
  • Viewer Access Control — block traffic according to geographic area
  • Content Protection with Signed URLs — prevent URL copying and sharing
  • Built-in Failover and origin failover for high availability
Performance Architecture
  • Multi-layer caching architecture — aggregates traffic from CloudFront edge locations
  • HTTP/3 (QUIC/UDP) multiplexing for faster connections
  • Persistent connections to origins — no repeated TCP handshakes, maintaining TCP traffic efficiently
  • CloudFront Functions — redirect users to region-specific endpoints

Session 5 – Build a Product in 36 Hours with LotusHacks

A speaker shared their hackathon experience at LotusHacks:

  • Built an app that allows direct interaction with UI components for live editing through 3 AI agents.
  • The app provides edit history review, allowing users to see and revert past changes.
  • Shared lessons learned from the intense 36-hour product-building sprint.

Session 6 – Non-determinism of Deterministic LLM Settings

A technical session on how LLMs work internally:

  • How LLMs generate text: The model ranks scores for each word in the prompt to select the most suitable next token from its vocabulary.
  • Inference optimization methods: Current LLM providers use inference optimizations to deliver cost-effective pricing plans while maintaining output quality.

Session 7 – Enterprise-grade Multi-Agent System: Startup Credit Scoring

A case study on building a multi-agent system for evaluating startup creditworthiness:

The Problem
  • Evaluating credit scores for startup businesses with limited historical data.
  • All existing startup data cannot be fed into current models — the challenge is generating reliable AI outputs within a fixed budget.
  • Every tool used and every line of code generated must deliver measurable value.
Agent Architecture

The system uses a Manager agent coordinating specialized sub-agents:

  • Financial Analyst — evaluate financial health
  • Market Analyst — assess market positioning
  • Team Evaluator — review founding team capabilities
  • Risk Assessor — identify potential risks
  • Compliance — ensure regulatory adherence
Why Multi-Agent Architecture Works
  • Specialized expertise — each agent focuses on its domain
  • Checks & balances — agents cross-validate each other
  • Parallel processing — multiple evaluations run simultaneously
  • Auditability — each agent’s reasoning is traceable
  • Scalability — add new agents as needs grow
  • Fault tolerance — system remains functional even if one agent fails
Key Principle

Build systems that are safe, reliable, and user-supportive.

Lessons Learned & Personal Takeaways

  • Personal branding matters: In a rapidly evolving market, building and showcasing your own unique value is essential for career growth.
  • Context engineering > prompt engineering: The quality of AI output depends heavily on providing focused, relevant context — not just good prompts. Avoid over-explaining or flooding with irrelevant data.
  • Obsidian as a knowledge system: Using structured personal knowledge bases like Obsidian can dramatically improve the context you provide to AI tools.
  • CloudFront is more than a CDN: The new pricing model, VPC Origin cloaking, signed URLs, and HTTP/3 support make CloudFront a comprehensive edge security and performance platform.
  • Multi-agent systems for complex problems: Decomposing a complex problem (like credit scoring) into specialized agents with checks and balances produces more reliable, auditable results than a single monolithic AI approach.
  • LLM internals awareness: Understanding how token scoring and inference optimization work helps make better decisions when choosing providers and pricing plans.
