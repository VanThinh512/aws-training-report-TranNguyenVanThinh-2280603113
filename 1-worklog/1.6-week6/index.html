Week 7 Achievements:

  • Successfully scaffolded and ran 3 FastAPI microservices with independent responsibilities:
    • auth-service – registration, login, RS256 JWT issuing with tenant claims
    • tenant-service – corporate tenant accounts, subscriptions, subdomains
    • hr-service – core HR domain (departments, employees, attendance, leave) with Pool Model tenant data separation

  • Built the ReactJS (Vite) frontend as a Single Page Application:

    • Login, Dashboard, Employee Directory, Attendance, and Leave Request pages
    • Client-side routing and an Axios-based API layer talking to the gateway

  • Configured the complete local development environment:

    • .env with database URLs and service ports
    • RS256 asymmetric JWT key pair (private key signs, public key verifies)
    • init.sql creating and seeding 3 independent MySQL databases

  • Deployed the full stack with Docker Compose:

    • Dockerfiles for every microservice and the frontend
    • Nginx API Gateway handling routing, CORS, SSL termination, and request tracing (X-Correlation-ID)
    • MySQL exposed on host port 3307 to avoid local port conflicts
    • One-command startup (docker-compose up -d --build) bringing up the entire client–server system

  • Verified the end-to-end flow through the gateway and confirmed inter-service communication works.

  • Delegated the remaining feature work to team members (see below).

Task Assignment for Team Members (Remaining APIs):

1. Authentication Service (auth-service) — Workspace Users management group (not yet implemented):

  • GET /api/v1/auth/tenants/users – List all users belonging to the current tenant.
  • PUT /api/v1/auth/tenants/users/{id}/role – Update a member’s role (Owner, Admin, Member).
  • DELETE /api/v1/auth/tenants/users/{id} – Remove/revoke a member’s access from the tenant.

2. HR Service (hr-service) — Employee update & delete operations (not yet implemented):

  • PUT /api/v1/hr/employees/{id} – Update an employee’s profile/position.
  • DELETE /api/v1/hr/employees/{id} – Remove an employee from the tenant’s system.

3. Tenant Service (tenant-service) — Self-service tenant configuration (not yet implemented):

  • GET /api/v1/tenants/my-tenant – Allow a logged-in user to view their own company’s information.
  • PUT /api/v1/tenants/my-tenant – Allow the Owner to update the company logo and name.
