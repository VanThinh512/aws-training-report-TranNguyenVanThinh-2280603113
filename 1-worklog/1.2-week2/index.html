Week 2 Achievements:

  • Access Management (AWS IAM)

    • Configured detailed access controls for AWS resources using IAM Groups, Users, and Customer Managed Policies.
    • Implemented secure temporary credentials and delegated access using IAM Roles rather than permanent security keys.
    • Applied the principle of least privilege across administrative and user-level permissions.
    • 💡 Pro Tips & Notes:
      • Always use IAM Roles for temporary permissions and set session timeouts to limit exposure.
      • Monitor identity access and API activities continuously using AWS CloudTrail.

  • Secure Application Access (IAM Roles on EC2)

    • Replaced hardcoded IAM Access Keys and Secret Access Keys in application code with temporary credentials from an IAM Role.
    • Provisioned an EC2 instance and S3 bucket to test secure, credential-less programmatic uploads.
    • 💡 Pro Tips & Notes:
      • Hardcoded credentials pose a high security risk. Always assign IAM Roles to EC2 instances for secure S3/AWS service access.

  • Static Website Hosting (Amazon S3 & CloudFront)

    • Configured Amazon S3 static website hosting, utilizing S3 Standard storage with 99.999999999% durability.
    • Integrated S3 hosting with Amazon CloudFront using Origin Access Control (OAC) to secure the origin bucket.
    • Enabled S3 Versioning to preserve historical object states and configured Cross-Region Replication (CRR) from Singapore for geographic redundancy.
    • 💡 Pro Tips & Notes:
      • Keep S3 Block Public Access enabled and route traffic exclusively through CloudFront to enforce HTTPS and caching.
      • Use S3 Lifecycle Policies to automatically transition or expire old object versions to control storage costs.

  • Network Infrastructure (Amazon VPC)

    • Designed and provisioned custom VPCs with public subnets (routed to IGW) and private subnets (routed to NAT Gateway) across multiple AZs.
    • Configured stateful Security Groups (instance-level) and stateless Network ACLs (subnet-level) for multi-layered network security.
    • Configured Site-to-Site VPN with dual redundant IPSec tunnels and enabled VPC Flow Logs for traffic analysis.
    • 💡 Pro Tips & Notes:
      • AWS reserves 5 IP addresses in each subnet (first 4 and last 1) for internal networking.
      • Use NAT Gateways instead of NAT Instances in production for automatic scaling and lower administrative overhead.

  • Compute Services (Amazon EC2)

    • Provisioned and connected to Windows Server 2025 (via RDP) and Amazon Linux 2023 (via SSH/Session Manager).
    • Deployed a Node.js CRUD User Management application under LAMP (Linux) and XAMPP (Windows) environments.
    • Created EBS snapshots, custom AMIs, and applied IAM governance policies to restrict EC2 regions, instance types, and volume categories.
    • 💡 Pro Tips & Notes:
      • Terminate unused instances and release associated Elastic IPs to prevent unexpected charges.
      • Use AWS Systems Manager Session Manager for secure instance access without opening inbound SSH/RDP ports.

  • Managed Databases (Amazon RDS)

    • Selected managed database engines (MySQL, PostgreSQL, Aurora) in Multi-AZ DB Subnet Groups.
    • Configured Multi-AZ synchronous replication with automatic failover (60-120 seconds) and asynchronous Read Replicas for read-heavy workloads.
    • Established backup management using automated point-in-time recovery (up to 35 days) and manual DB snapshots.
    • 💡 Pro Tips & Notes:
      • Automated snapshots are performed on standby Multi-AZ instances to prevent primary database latency.
      • To encrypt an existing unencrypted database, copy its snapshot, apply KMS encryption to the copy, and restore from it.

  • Compute Cost Optimization (Amazon Lightsail)

    • Provisioned simplified, low-cost application stacks (WordPress, Akaunting) and high-availability database instances on Lightsail.
    • Minimized host attack surface by removing port 22 (SSH) from both IPv4 and IPv6 instance firewalls.
    • Configured manual snapshots, automatic daily snapshots, and resource alarms (e.g. CPU utilization).
    • 💡 Pro Tips & Notes:
      • Manual snapshots are persistent and not automatically deleted when the parent instance is terminated; clean them up manually to avoid storage costs.

  • Application Autoscaling (EC2 Auto Scaling & ELB)

    • Created custom AMIs and configured Launch Templates to boot uniform application servers across multiple AZs.
    • Provisioned Application Load Balancers (ALBs) and Target Groups, integrating them with Auto Scaling Groups (ASGs).
    • Evaluated scaling strategies (Manual, Scheduled, Dynamic, Predictive) and validated dynamic scaling by load testing with Paessler WebStress.
    • 💡 Pro Tips & Notes:
      • Integrate ELB with AWS Certificate Manager (ACM) to offload SSL/TLS encryption (SSL termination) from target servers.
      • Combine predictive scaling with target tracking dynamic scaling to proactively handle cyclical traffic spikes.

  • Observability (Amazon CloudWatch)

    • Collected performance metrics and centralized application/system logs under CloudWatch Log Groups.
    • Created custom metric alarms to trigger automated notifications and instance recovery actions.
    • Built unified CloudWatch Dashboards to monitor overall application and infrastructure health.
    • 💡 Pro Tips & Notes:
      • Enable Container Insights to automatically collect and aggregate compute metrics (CPU, memory, disk, network) for ECS, EKS, and Fargate.

  • Hybrid DNS Integration (Amazon Route 53)

    • Explored Route 53 Resolver capabilities including private hosted zones (PHZ) and recursive query resolution.
    • Configured Inbound Endpoints (resolving AWS domains from local) and Outbound Endpoints (resolving local domains from AWS).
    • Created Resolver Rules to conditionally forward queries for specific domain namespaces to local DNS servers.
    • 💡 Pro Tips & Notes:
      • Distribute DNS endpoints across multiple Availability Zones to prevent single points of failure in DNS routing.

  • Command Line Administration (AWS CLI)

    • Configured the AWS CLI with default/named profiles on remote EC2 instances.
    • Executed API operations and parsed text-based output using Unix utilities (grep, awk, sed).
    • 💡 Pro Tips & Notes:
      • Use the ’text’ output format in shell scripts for easy integration and processing with command-line utilities.
      • Always use IAM Roles instead of static credentials (access keys) to grant secure access.
