Kết quả đạt được tuần 4:

  • Triển khai ứng dụng container với Docker & Dịch vụ AWS

    • Nghiên cứu các thành phần cốt lõi của môi trường container bao gồm Docker Hub, Nginx reverse proxy, Amazon ECR, Amazon RDS và Amazon EC2.
    • Tìm hiểu luồng xử lý từ client đến cơ sở dữ liệu: điều hướng traffic qua Internet Gateway và Nginx, tương tác giữa các frontend/backend container, và lưu trữ dữ liệu bền vững trên RDS.
    • Khảo sát phương pháp triển khai container cục bộ (local) so với việc vận hành container trên máy chủ ảo đám mây (EC2).
    • Nghiên cứu các tính năng của RDS như DB subnet groups, Multi-AZ, read replicas và tự động mở rộng dung lượng ổ cứng.
    • Tìm hiểu cách điều phối ứng dụng bằng Docker Compose cùng quy trình đẩy/kéo ảnh Docker từ ECR và Docker Hub.

  • Điều phối Container với Amazon Elastic Container Service (Amazon ECS)

    • Nghiên cứu kiến trúc Amazon ECS và các thành phần cốt lõi: Clusters, Task Definitions, Services, cùng môi trường vận hành container (EC2 vs. Fargate).
    • Tìm hiểu Task Definition như một bản thiết kế định nghĩa ảnh Docker, phân bổ CPU/Memory, chế độ mạng, cấu hình logs và các vai trò IAM thực thi (execution role).
    • Khảo sát cơ chế tự động mở rộng tài nguyên và ứng dụng: Cluster Scaling (mở rộng máy chủ EC2/Fargate) và Service Scaling (điều chỉnh số lượng task chạy qua CloudWatch).
    • Nghiên cứu các chiến lược cập nhật ứng dụng: Rolling Deployments (thay thế container dần dần) và Blue/Green Deployments (vận hành hai môi trường song song với chuyển hướng traffic qua CodeDeploy và ALB).
    • Tìm hiểu quá trình chuẩn bị hạ tầng bao gồm thiết lập CodeDeploy execution roles, NAT Gateways, Route Tables, và Application Load Balancers (ALB) để điều phối lưu lượng đến các dịch vụ frontend/backend.

  • Tích hợp liên tục, Bàn giao liên tục và Giám sát (CI/CD, Container Insights, FireLens)

    • Tìm hiểu quy trình tích hợp liên tục (CI) và bàn giao liên tục (CD) nhằm tự động hóa việc biên dịch, kiểm thử và triển khai mã nguồn.
    • So sánh quy trình tự động hóa CI/CD trên các nền tảng GitHub Actions, GitLab CI/CD, và AWS CodePipeline (tích hợp CodeBuild và CodeDeploy).
    • Nghiên cứu các giai đoạn tự động hóa đường ống: kích hoạt qua git tag, build ảnh Docker, push lên ECR/Docker Hub, cập nhật Task Definition, và cập nhật ECS Service (qua CodeDeploy blue/green hoặc rolling).
    • Tìm hiểu quy trình tự động hóa triển khai ứng dụng lên các máy chủ Amazon EC2 bằng bộ công cụ phát triển của AWS (CodePipeline, CodeBuild, và CodeDeploy).
    • Nghiên cứu vòng đời của AWS CodeCommit (ngừng hỗ trợ đối với khách hàng mới từ tháng 7/2024) và quy trình chuyển đổi dữ liệu kho lưu trữ sang các nền tảng khác (GitHub, GitLab, Bitbucket) thông qua kỹ thuật git mirroring.
    • Tìm hiểu vai trò của AWS CodeBuild trong biên dịch và kiểm thử mã nguồn trên các build servers động, và AWS CodeDeploy (sử dụng CodeDeploy Agent trên EC2) để tự động cài đặt ứng dụng.
    • Khảo sát cơ chế lưu trữ bảo mật các artifact của đường ống dẫn trên Amazon S3 với mã hóa dữ liệu tĩnh được quản lý bằng khóa AWS KMS.
    • Nghiên cứu giám sát hiệu năng container bằng Amazon CloudWatch Container Insights để theo dõi CPU, RAM, lưu lượng mạng và trạng thái các task replicas.
    • Tìm hiểu kiến trúc quản lý và định tuyến logs của container sử dụng AWS FireLens để điều hướng logs tới các đích lưu trữ bên ngoài như Amazon S3 và CloudWatch Logs.

  • Lưu trữ lai & Chia sẻ tệp tin (AWS Storage Gateway & FSx)

    • Nghiên cứu kiến trúc AWS Storage Gateway, tập trung vào cách File Storage Gateway hoạt động như một cầu nối đồng bộ hóa dữ liệu giữa môi trường on-premises và Amazon S3.
    • Khảo sát cấu hình triển khai ảo hóa của thiết bị Storage Gateway trên các nền tảng ảo hóa (VMware ESXi, Microsoft Hyper-V, Linux KVM) hoặc sử dụng các bản EC2 AMI cấu hình sẵn.
    • Tìm hiểu quy trình tạo, cấu hình các File Share (NFS/SMB) và mount chúng vào máy chủ cục bộ hoặc on-premises nhằm mở rộng dung lượng lưu trữ tích hợp đám mây không giới hạn.
    • Nghiên cứu kiến trúc Amazon FSx cho Windows File Server bao gồm máy chủ tệp tin EC2 sử dụng giao thức SMB, cơ chế đồng bộ dữ liệu S3, cô lập mạng bằng VPC và kết nối qua ENI.
    • Tìm hiểu cách tích hợp hệ thống tệp tin với Windows Active Directory và cấu hình triển khai Multi-AZ (SSD hoặc HDD) để đạt tính sẵn sàng cao.
    • Khảo sát các tính năng quản trị nâng cao của máy chủ tệp tin: chống trùng lặp dữ liệu (data deduplication), chụp ảnh bóng (shadow copies - VSS), quản lý phiên kết nối người dùng, kiểm soát hạn ngạch lưu trữ, và chia sẻ tệp tin duy trì truy cập liên tục (Continuous Access - CA).

  • Xây dựng Data Lake & Kiến trúc NoSQL (DynamoDB)

    • Tìm hiểu các khái niệm và đặc tính cốt lõi của một Data Lake trên AWS bao gồm thu thập dữ liệu thô, cho phép nhiều người dùng đồng thời khám phá và hỗ trợ các mô hình truy cập linh hoạt.
    • Nghiên cứu luồng phân phối dữ liệu serverless sử dụng Amazon Kinesis Firehose Delivery Streams và mô phỏng tạo dữ liệu mẫu thông qua Kinesis Data Generator.
    • Khảo sát kiến trúc serverless ETL của AWS Glue: thu thập siêu dữ liệu qua Glue Crawlers, quản lý cấu trúc bảng trong Glue Data Catalog và sinh mã Python Spark tự động cho các tác vụ biến đổi dữ liệu.
    • Tìm hiểu môi trường biến đổi và xử lý dữ liệu sử dụng các sổ tay SageMaker Notebooks để chuẩn bị dữ liệu cho các phân tích tiếp theo.
    • Nghiên cứu cách truy vấn SQL tức thì trên S3 bằng Amazon Athena (sử dụng Presto engine chuẩn ANSI SQL) và kiến trúc trực quan hóa dữ liệu bằng Amazon QuickSight (thông qua Data sources, Datasets, Analyses, Visuals, và Dashboards).
    • Nghiên cứu các tính năng của Amazon DynamoDB như một cơ sở dữ liệu NoSQL dạng key-value và tài liệu được quản lý hoàn toàn, cung cấp độ trễ phản hồi dưới 10 mili-giây ở mọi quy mô.
    • Tìm hiểu các phương pháp thiết kế tốt nhất (best practices), các mẫu thiết kế nâng cao, và các đánh đổi kiến trúc (như thiết kế Khóa chính, local/global secondary indexes) sử dụng AWS CLI và Console.
    • Nghiên cứu cấu hình cơ sở dữ liệu toàn cầu sử dụng DynamoDB Global Tables để hỗ trợ đồng bộ dữ liệu đa vùng với độ trễ thấp.
    • Khảo sát hoạt động thu thập thay đổi dữ liệu bằng DynamoDB Streams tích hợp với AWS Lambda để xây dựng các kiến trúc hướng sự kiện serverless.
    • Nghiên cứu luồng tích hợp zero-ETL từ DynamoDB để liên tục lập chỉ mục dữ liệu sang Amazon OpenSearch Service, hỗ trợ các câu lệnh truy vấn ngôn ngữ tự nhiên được cung cấp bởi Amazon Bedrock.

  • Các chiến lược tối ưu hóa chi phí AWS

    • Nghiên cứu AWS Savings Plans (loại Compute và EC2 Instance) như một mô hình giá linh hoạt đòi hỏi cam kết chi tiêu theo giờ trong 1 hoặc 3 năm, đi kèm 3 lựa chọn thanh toán (All, Partial, hoặc No Upfront).
    • Tìm hiểu cách sử dụng Savings Plan Recommendations, các chu kỳ xem xét dữ liệu lịch sử (lookback periods), lập lịch kích hoạt kế hoạch trong tương lai và chia sẻ tiết kiệm chi phí qua AWS Organizations.
    • So sánh Standard RIs và Convertible RIs về tính linh hoạt (cho phép đổi instance family, OS, region, AZ), mức giảm giá, và quy trình bán lại các Standard RIs trên RI Marketplace.
    • Nghiên cứu Reserved DB Instances của Amazon RDS bao gồm các yêu cầu cấu hình khớp (DB engine, instance class, Multi-AZ) để áp dụng chính sách giảm giá và hành vi khi kích thước DB instance tự động mở rộng.
    • Nghiên cứu việc tối ưu hóa kích cỡ máy chủ EC2 (Right-sizing) sử dụng CloudWatch Agent để theo dõi các chỉ số tùy chỉnh như dung lượng RAM bên cạnh các chỉ số mặc định (CPU, Network In/Out).
    • Tìm hiểu quy trình triển khai CloudWatch Agent trên máy chủ EC2 bao gồm cấu hình phân quyền IAM (CloudWatchAgentServerPolicy) và cài đặt qua Config Wizard.
    • Nghiên cứu công cụ Amazon EC2 Resource Optimization (tích hợp trong AWS Cost Explorer) giúp phân tích lịch sử sử dụng tài nguyên trong 14 ngày gần nhất để đưa ra đề xuất: Downsizing (hạ cấu hình nếu CPU tối đa ở mức 1% - 40%) hoặc Terminate (tắt máy chủ nhàn rỗi nếu CPU tối đa <= 1%).
    • Nắm vững các nguyên tắc tốt nhất về định cỡ tài nguyên phù hợp: thực hiện tối ưu trước khi dịch chuyển hệ thống, đo lường kỹ lưỡng, bắt đầu với môi trường thử nghiệm và xử lý riêng biệt các dòng máy chủ burstable (T-types).

  • Trực quan hóa & Phân tích chi phí (AWS Glue & Athena)

    • Khảo sát các công cụ theo dõi và trực quan hóa chi phí sử dụng tài nguyên AWS bao gồm AWS Cost Explorer và thiết lập cảnh báo vượt ngưỡng qua AWS Budgets.
    • Nghiên cứu cách sử dụng AWS Glue Crawlers để lập danh mục dữ liệu cho các báo cáo chi phí và sử dụng (Cost and Usage Reports - CUR) lưu trữ trên Amazon S3.
    • Tìm hiểu cách thực thi các truy vấn SQL ad-hoc thông qua Amazon Athena để phân tích chuyên sâu các tập dữ liệu chi phí đã được lập danh mục.
