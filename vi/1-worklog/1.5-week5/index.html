Mục tiêu tuần 5:

  • Chiến lược đóng gói container và triển khai ứng dụng
  • Đường ống tích hợp và bàn giao liên tục (CI/CD)
  • Giải pháp lưu trữ đám mây lai (hybrid storage) và chia sẻ tệp tin
  • Kiến trúc Data Lake và thiết kế cơ sở dữ liệu NoSQL
  • Mô hình tối ưu hóa chi phí và điều chỉnh kích cỡ tài nguyên compute (right-sizing)
  • Trực quan hóa chi phí, danh mục dữ liệu (data cataloging) và truy vấn phân tích

Kết quả đạt được tuần 5:

  • Triển khai ứng dụng dạng Container với Docker & các dịch vụ AWS

    • Tìm hiểu các thành phần cốt lõi của công nghệ đóng gói container bao gồm Docker Hub, Nginx reverse proxy, Amazon ECR, Amazon RDS và Amazon EC2.
    • Khảo sát luồng tương tác từ client đến database: định tuyến lưu lượng qua Internet Gateway và Nginx, giao tiếp giữa frontend/backend container, và lưu trữ dữ liệu bền vững trên RDS.
    • Nghiên cứu phương pháp triển khai container cục bộ so với việc chạy container trên các máy chủ ảo đám mây (EC2).
    • Tìm hiểu các tính năng của RDS như DB subnet groups, Multi-AZ deployments, read replicas, và tự động mở rộng dung lượng lưu trữ.
    • Hiểu cách điều phối ứng dụng bằng Docker Compose và quy trình đẩy/kéo ảnh container từ Amazon ECR và Docker Hub.

  • Điều phối Container với Amazon Elastic Container Service (ECS)

    • Nghiên cứu kiến trúc Amazon ECS và các thành phần chính: Clusters, Task Definitions, Services, và các môi trường chạy container (EC2 vs. Fargate).
    • Tìm hiểu Task Definitions dưới dạng các bản thiết kế định nghĩa ảnh Docker, phân bổ CPU/RAM, chế độ mạng, cấu hình log, và các IAM execution roles.
    • Khảo sát các cơ chế mở rộng ứng dụng và tài nguyên: mở rộng Cluster (thay đổi tài nguyên instances/Fargate) và mở rộng Service (điều chỉnh số lượng task đang chạy qua CloudWatch).
    • Tìm hiểu chiến lược cập nhật ứng dụng: Rolling Deployments (thay thế container dần dần) và Blue/Green Deployments (chuyển đổi lưu lượng giữa hai môi trường song song qua CodeDeploy và ALB).
    • Nghiên cứu các bước chuẩn bị hạ tầng bao gồm CodeDeploy execution roles, NAT Gateways, Route Tables, và Application Load Balancers (ALB) để định tuyến dịch vụ.

  • Tích hợp, Bàn giao liên tục và Giám sát (CI/CD, Container Insights, FireLens)

    • Tìm hiểu các khái niệm cốt lõi của Tích hợp liên tục (CI) và Bàn giao liên tục (CD) để tự động hóa quy trình biên dịch, kiểm thử và triển khai mã nguồn.
    • So sánh quy trình CI/CD và đường ống tự động hóa trên GitHub Actions, GitLab CI/CD, và AWS CodePipeline (tích hợp CodeBuild và CodeDeploy).
    • Hiểu các giai đoạn của đường ống tự động: kích hoạt qua git tags, build Docker images, đẩy lên ECR/Docker Hub, cập nhật Task Definitions và cập nhật ECS Services.
    • Nghiên cứu quy trình tự động triển khai ứng dụng lên các máy chủ Amazon EC2 sử dụng bộ công cụ dành cho nhà phát triển của AWS (CodePipeline, CodeBuild, và CodeDeploy).
    • Tìm hiểu tình trạng vòng đời của CodeCommit (không còn hỗ trợ cho khách hàng mới từ tháng 7 năm 2024) và quy trình di chuyển dữ liệu kho lưu trữ sang các nhà cung cấp SCM khác (GitHub, GitLab, Bitbucket) sử dụng kỹ thuật git mirroring.
    • Hiểu vai trò của AWS CodeBuild trong việc biên dịch và kiểm thử mã nguồn trên các máy chủ build động, và AWS CodeDeploy (sử dụng CodeDeploy Agent trên EC2) trong tự động hóa cài đặt ứng dụng.
    • Khảo sát việc lưu trữ an toàn các artifact của đường ống trên Amazon S3 với cơ chế mã hóa tĩnh do khóa AWS KMS quản lý.
    • Nghiên cứu giám sát hiệu năng container bằng cách sử dụng Amazon CloudWatch Container Insights để theo dõi CPU, bộ nhớ, lưu lượng mạng, và trạng thái của các task replica.
    • Tìm hiểu quản lý và định tuyến nhật ký (log) sử dụng AWS FireLens để chuyển log container đến các đích lưu trữ bên ngoài như Amazon S3 và CloudWatch Logs.

  • Lưu trữ đám mây lai & Chia sẻ tệp tin (AWS Storage Gateway & FSx)

    • Tìm hiểu kiến trúc AWS Storage Gateway, tập trung vào cách File Storage Gateway đóng vai trò làm cầu nối đồng bộ dữ liệu giữa môi trường on-premises và Amazon S3.
    • Nghiên cứu cấu hình triển khai ảo hóa của Storage Gateway trên các nền tảng (VMware ESXi, Microsoft Hyper-V, Linux KVM) hoặc sử dụng EC2 AMI cấu hình sẵn.
    • Hiểu quy trình tạo và cấu hình File Shares (NFS/SMB) và kết nối (mount) chúng vào các máy chủ cục bộ hoặc on-premises.
    • Nghiên cứu kiến trúc Amazon FSx cho Windows File Server, bao gồm các file server chạy trên EC2 sử dụng giao thức SMB, sao lưu dữ liệu S3, cô lập VPC và kết nối mạng dựa trên ENI.
    • Tìm hiểu tích hợp miền thông qua Windows Active Directory và thiết lập hệ thống tệp tin tính sẵn sàng cao sử dụng cấu hình Multi-AZ với ổ cứng SSD hoặc HDD.
    • Khảo sát các tính năng quản trị nâng cao của file server: khử trùng lặp dữ liệu (data deduplication), bản sao bóng shadow copies (VSS), quản lý phiên người dùng, hạn ngạch lưu trữ, và chia sẻ tệp tin Continuous Access (CA).

  • Xây dựng Data Lake & Kiến trúc NoSQL (DynamoDB)

    • Tìm hiểu các khái niệm và đặc tính cốt lõi của Data Lake trên AWS, bao gồm thu thập dữ liệu thô, khám phá dữ liệu đa người dùng và các mô hình truy cập linh hoạt.
    • Nghiên cứu đường ống thu nhận dữ liệu serverless sử dụng Amazon Kinesis Firehose Delivery Streams và tạo dữ liệu mẫu qua Kinesis Data Generator.
    • Tìm hiểu kiến trúc ETL serverless của AWS Glue: lập danh mục siêu dữ liệu với Glue Crawlers, quản lý schema trong Glue Data Catalog, và tạo các job Python Spark để chuyển đổi dữ liệu.
    • Khảo sát môi trường biến đổi dữ liệu sử dụng SageMaker Notebooks để chuẩn bị các tập dữ liệu cho phân tích hạ nguồn.
    • Nghiên cứu thực hiện truy vấn SQL ad-hoc trên S3 bằng Amazon Athena (sử dụng công cụ ANSI SQL Presto) và kiến trúc trực quan hóa dữ liệu qua Amazon QuickSight (Data sources, Datasets, Analyses, Visuals, và Dashboards).
    • Nghiên cứu các đặc tính của Amazon DynamoDB như một cơ sở dữ liệu tài liệu và key-value được quản lý hoàn toàn, có độ trễ mili-giây ở mọi quy mô.
    • Tìm hiểu các thực hành thiết kế mô hình dữ liệu NoSQL, các mẫu thiết kế nâng cao, và đánh đổi kiến trúc (như khóa chính và chỉ mục) thông qua AWS CLI và Console.
    • Nghiên cứu cấu hình cơ sở dữ liệu serverless toàn cầu sử dụng DynamoDB Global Tables để sao chép đa vùng và giảm thiểu độ trễ truy cập.
    • Tìm hiểu cơ chế Change Data Capture (CDC) sử dụng DynamoDB Streams tích hợp với AWS Lambda để xây dựng kiến trúc hướng sự kiện serverless.
    • Khảo sát các mô hình tích hợp zero-ETL hiện đại, chẳng hạn như liên tục lập chỉ mục danh mục DynamoDB sang Amazon OpenSearch Service để hỗ trợ các truy vấn ngôn ngữ tự nhiên thông qua Amazon Bedrock.

  • Chiến lược tối ưu hóa chi phí AWS

    • Tìm hiểu AWS Savings Plans (loại Compute và EC2 Instance) dưới dạng mô hình giá linh hoạt yêu cầu cam kết số tempo sử dụng theo giờ trong 1 hoặc 3 năm với các lựa chọn thanh toán (Trả trước toàn bộ, Trả trước một phần, hoặc Không trả trước).
    • Nghiên cứu các khuyến nghị Savings Plan, khoảng thời gian phân tích lịch sử sử dụng (lookback periods), lập lịch mua gói trong tương lai, và chia sẻ hạn mức tối ưu chi phí trong AWS Organizations.
    • So sánh Standard Reserved Instances và Convertible Reserved Instances (RIs) về mức độ linh hoạt (thay đổi instance family, hệ điều hành, region, AZ), mức giảm giá, và việc bán lại Standard RI trên thị trường RI Marketplace.
    • Nghiên cứu Reserved DB Instances trong Amazon RDS, bao gồm các yêu cầu cấu hình khớp nhau (DB engine, instance class, Multi-AZ) để áp dụng giảm giá và hành vi khi kích thước instance tự động thay đổi.
    • Nghiên cứu điều chỉnh kích cỡ EC2 (right-sizing) sử dụng Amazon CloudWatch Agent để giám sát các số liệu tùy chỉnh như mức sử dụng bộ nhớ bên cạnh các thông số mặc định (mức sử dụng CPU, Network In/Out).
    • Tìm hiểu việc triển khai CloudWatch Agent trên các node EC2, bao gồm cấu hình IAM policy (CloudWatchAgentServerPolicy) và trình hướng dẫn cài đặt wizard.
    • Nghiên cứu tính năng Amazon EC2 Resource Optimization (qua AWS Cost Explorer) để phân tích lịch sử 14 ngày và đưa ra các khuyến nghị tối ưu: Downsizing (giảm kích cỡ cho tài nguyên sử dụng thấp, CPU 1%–40%) hoặc Terminating (tắt tài nguyên nhàn rỗi, CPU <=1%).
    • Hiểu các thực hành tốt nhất về điều chỉnh kích cỡ: tối ưu hóa trước khi di chuyển, đo lường kỹ lưỡng khối lượng công việc, bắt đầu với môi trường thử nghiệm (non-production), và quản lý riêng biệt các dòng instance burstable (T-types).

  • Trực quan hóa & Phân tích chi phí (AWS Glue & Athena)

    • Nghiên cứu các công cụ theo dõi và trực quan hóa chi phí bao gồm AWS Cost Explorer và AWS Budgets để thiết lập cảnh báo ngưỡng ngân sách.
    • Tìm hiểu việc phân mục dữ liệu sử dụng AWS Glue Crawlers để lập chỉ mục các báo cáo chi phí và sử dụng Cost and Usage Reports (CUR) lưu trữ trên Amazon S3.
    • Nghiên cứu phân tích truy vấn SQL ad-hoc sử dụng Amazon Athena để truy vấn và phân tích các tập dữ liệu chi phí đã được lập danh mục.
